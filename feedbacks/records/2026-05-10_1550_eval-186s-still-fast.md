# Evaluation Still Too Fast

## Date

2026-05-10 15:50

## Run ID

`2026-05-10_1550_eval-186s-still-fast`

## Observation

The `seq_a:256,seq_b:256,seq_c:512,seq_d:448` evaluation completed with
platform inference time `186.62s`.

This is still much faster than the previous useful baseline runtime around
`1200s`, so sequence-length tuning alone is not enough to spend the available
budget.

## Diagnosis

The evaluator is now efficient enough that even near-full long-domain windows
leave most of the inference-time budget unused. Also, the dataset currently
keeps the prefix of each sequence window, so a recency-focused view may recover
signal that is lost by prefix truncation.

## Action

Add `recent_blend` test-time augmentation to `New_baseline/infer.py`:

- `EVAL_TTA_MODE=recent_blend`
- `EVAL_TTA_WEIGHT=0.12`
- `EVAL_TTA_CROP_LENS=seq_a:128,seq_b:128,seq_c:256,seq_d:256`
- `EVAL_TTA_REPEAT=5`

The blend creates a recency-focused in-memory crop and averages its logits into
the full-window logits. `EVAL_TTA_REPEAT` is a runtime dial; it repeats the same
recent view to move closer to the platform budget without changing predictions
after the first recent pass.

## Next Sweep

Submit once with the default TTA settings. If runtime is still too low, increase
`EVAL_TTA_REPEAT` to `8`. If it is close to timeout, reduce it to `3`.
