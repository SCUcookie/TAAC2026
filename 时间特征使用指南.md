# 时间特征使用指南

## 📊 概述

本文档介绍了为腾讯算法大赛模型新增的时间差特征系统，旨在通过挖掘用户行为的时间模式来提升推荐效果。

## 🎯 核心优势

基于数据分析发现的时间模式：
- **65.4%** 的用户行为集中在1-10分钟间隔
- **平均点击间隔**: 4.7分钟，中位数2.7分钟
- **明显时间周期性**: 工作日vs周末，一天中不同时段
- **高用户参与度**: 98.29%用户有点击行为

## 🚀 快速开始

### 1. 训练模型（启用时间特征）

```bash
# 使用默认参数训练（时间特征默认启用）
python -u main.py

# 显式启用时间特征
python -u main.py --enable_time_features

# 自定义时间衰减因子
python -u main.py --enable_time_features --time_decay_factor 0.05
```

### 2. 推理（保持训推一致性）

```bash
# 推理时确保与训练参数一致
python infer.py --enable_time_features --time_decay_factor 0.1
```

### 3. 禁用时间特征（对比测试）

```bash
# 训练时禁用
python -u main.py --no-enable_time_features

# 推理时禁用
python infer.py --no-enable_time_features
```

## 📈 时间特征详解

### 用户级时间特征（写入user_feat）

| 特征名 | 描述 | 取值范围 |
|--------|------|----------|
| `seq_avg_interval` | 序列平均时间间隔（分钟） | [0, ∞) |
| `seq_interval_variance` | 时间间隔方差 | [0, ∞) |
| `seq_click_frequency` | 点击频率（次/分钟） | [0, ∞) |
| `seq_temporal_consistency` | 时间一致性评分 | [0, ∞) |

### 物品级时间特征（写入item_feat）

#### 间隔特征
| 特征名 | 描述 | 取值范围 |
|--------|------|----------|
| `time_interval_minutes` | 距离上次行为间隔（分钟） | [0, ∞) |
| `time_interval_bucket` | 时间间隔分桶 | [0, 8] |
| `time_decay_weight` | 时间衰减权重 | [0, 1] |

#### 间隔类型（基于数据分析）
| 特征名 | 描述 | 条件 |
|--------|------|------|
| `is_immediate_click` | 即时点击 | ≤1分钟 (23.5%) |
| `is_short_interval` | 短间隔 | 1-10分钟 (65.4%) |
| `is_medium_interval` | 中间隔 | 10-60分钟 (11.0%) |
| `is_long_interval` | 长间隔 | >60分钟 (0.1%) |

#### 时间周期特征
| 特征名 | 描述 | 取值范围 |
|--------|------|----------|
| `hour` | 小时 | [0, 23] |
| `day_of_week` | 星期 | [0, 6] |
| `hour_sin`, `hour_cos` | 小时周期编码 | [-1, 1] |
| `day_sin`, `day_cos` | 星期周期编码 | [-1, 1] |

#### 活跃时段特征（基于数据发现）
| 特征名 | 描述 | 时间段 | 占比 |
|--------|------|--------|------|
| `is_morning_peak` | 上午高峰 | 6-12点 | 34.5% |
| `is_afternoon` | 下午时段 | 12-18点 | 28.2% |
| `is_evening` | 晚间高峰 | 18-24点 | 28.0% |
| `is_dawn` | 凌晨低峰 | 0-6点 | 9.2% |
| `is_weekend` | 周末标识 | 周六日 | 28.9% |

## 💡 最佳实践

### 1. 特征设计原则

- **训推一致性**: 时间特征仅在序列中使用，避免pos/neg样本带上时间特征
- **先用户后物品**: 时间特征先写入user_feat，再写入item_feat
- **避免泄露**: 不使用未来信息，严格按时间顺序处理

### 2. 参数调优建议

```bash
# 基础配置（推荐）
python main.py --enable_time_features --time_decay_factor 0.1

# 敏感调优（更关注近期行为）
python main.py --enable_time_features --time_decay_factor 0.2

# 平滑调优（更重视历史模式）
python main.py --enable_time_features --time_decay_factor 0.05
```

### 3. 模型架构建议

```python
# 模型自动适配时间特征维度
userdim = hidden_units * (sparse_feat_num + array_feat_num) + continual_feat_num + time_feat_num
itemdim = hidden_units * (sparse_feat_num + array_feat_num) + continual_feat_num + time_feat_num + emb_feat_dim
```
